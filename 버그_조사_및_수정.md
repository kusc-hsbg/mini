# 버그 조사 및 수정 기록

## 수집한 버그

- 방/워프 목록에서 같은 맵이 2개씩 보일 수 있음
  - 원인: 설정의 방 추가가 같은 `template_key`를 반복 생성할 수 있고, 기존 중복 방을 로드할 때 그대로 노출함.
  - 조치: 서버 로드 단계에서 같은 템플릿 방을 1개로 정리하고, 새 방 추가 시 같은 템플릿 중복 생성을 차단.

- 방 이동 직후 이전 맵 상태가 섞여 보일 수 있음
  - 원인: `liveMap` state가 최초 방 기준으로 초기화되고, 클라이언트 라우팅으로 `room` prop이 바뀔 때 엔진 생성 시점에 이전 state를 참조할 수 있음.
  - 조치: 방 prop 기반의 `resolvedRoomMap`을 별도로 계산하고, 엔진 부팅 시 해당 값을 사용하도록 수정.

- 맵 에디터로 저장한 커스텀 레이스/PK 맵에서 특수 동작이 사라질 수 있음
  - 원인: `resolveMap`이 `vehicle`, `race`, `pk` 필드를 복원하지 않음.
  - 조치: 커스텀 `map_data` 복원 시 세 필드도 유지.

- 상점이 월드 안 시설이 아니라 전체 입력을 막는 모달로만 열림
  - 원인: 상점 UI가 `Modal`에 고정되어 있고 맵 오브젝트 구매 흐름이 없음.
  - 조치: 우측 도킹 패널과 월드 내 진열대 구매/장착 흐름으로 변경.

- 스타홀 출구/전시 설계가 단순하고 이미지 기준과 다름
  - 원인: 기존 프리셋은 어두운 마루 위 NPC 나열과 텍스트 팝업 중심임.
  - 조치: 따뜻한 석재, 민트 발광 전시 프레임, 아치형 홀, 중앙 무대/카펫/식재 기준으로 재구성.

- 레이싱 트랙 안팎을 마음대로 넘나들 수 있음
  - 원인: 연석과 잔디/인필드가 모두 이동 가능 타일이라 트랙 가장자리가 실제 방벽 역할을 하지 못함.
  - 조치: 기본/테마 서킷의 외곽과 내곽에 연속 타이어 방벽을 추가하고, 스폰/카트 패드를 트랙 위로 보정.

- 레이싱 보스전이 기존 원거리 공격과 보스 접촉 판정에 의존함
  - 원인: 레이스 보스 피해가 일반 투사체 데미지로 계산되고, 보스가 플레이어 쪽으로 움직이는 호스트 시뮬레이션이 남아 있음.
  - 조치: 보스는 중앙 고정, HP는 15로 고정, 일반 화살은 미사일 요격만 가능, 아이템 박스 로켓만 보스에게 1 피해를 주도록 분리.

- 레이스 사망 후 즉시 일반 부활 흐름으로 돌아갈 수 있음
  - 원인: 레이스 사망도 PK 부활 타이머 구조를 공유할 수 있는 상태였음.
  - 조치: 유도 미사일 3회 피격 시 레이스 감옥 좌표로 이동하고, 보스 처치 또는 생존 플레이어 없음 조건에서만 해제.

- 자동차 소환 후 이동 속도가 걷기와 동일함
  - 원인: 이동 속도 계산이 `onBike`만 확인하고 상점 탈것의 `mounted`/`mount-sportscar` 상태를 반영하지 않음.
  - 조치: 스포츠카 탑승 상태에서 별도 이동 배속을 적용.

- 스타홀 전시 정보가 X 상호작용 모달에만 의존함
  - 원인: 전시대 근접 상태를 UI 정보 패널로 연결하지 않았음.
  - 조치: 스타홀에서 전시대 앞에 서면 좌측 정보 패널이 자동 표시되도록 추가.

- 상점 상품 수와 카테고리가 부족하고 데빌 윙이 남아 있음
  - 원인: 정적 카탈로그가 소수 상품 중심이고 춤 슬롯이 없음.
  - 조치: 데빌 윙 삭제, 대량 상품 생성 카탈로그 추가, 댄스 카테고리/장착 슬롯 추가.

- 레이스 방벽이 투사체까지 막아 보스 공격/로켓이 사라짐
  - 원인: 레이스 투사체도 캐릭터 이동용 solid grid를 그대로 사용함.
  - 조치: 레이스 방벽은 캐릭터 이동만 막고, 레이스 화살/로켓/보스 탄막은 방벽을 통과하도록 분리.

- 레이스 보스가 단일 패턴이라 보스별 차이가 없음
  - 원인: 모든 보스가 유도 미사일만 사용함.
  - 조치: 고슴도치 보스는 사방 가시/2스테이지 용암, 크라켄은 파도/먹물, 치킨은 알 탄막/고속 난사 패턴 추가.

- 게스트 지갑 때문에 React hydration 오류가 발생할 수 있음
  - 원인: 서버 첫 렌더는 기본 지갑값을 쓰고, 클라이언트 첫 렌더는 localStorage 지갑값을 즉시 읽어 텍스트가 달라짐.
  - 조치: 첫 렌더는 서버와 같은 기본값을 사용하고, 마운트 이후 localStorage 지갑을 반영하도록 변경.

- 탈것이 이모지에 의존하고 속도 체감이 약함
  - 원인: `drawMount`가 이모지 하나를 그리는 구조이고 이동 속도도 스포츠카만 일부 반영함.
  - 조치: 스포츠카/양탄자/짐승형/기계형/호버형 캔버스 렌더로 교체하고 탈것별 속도 배율 적용.

- 레이스 보스가 원형 도형처럼 보여 요구한 아이콘 보스와 다름
  - 원인: 보스 본체/자녀 렌더가 도형 중심이라 보스별 아이콘 정체성이 약함.
  - 조치: 레이스 보스는 중앙의 거대한 보스 아이콘으로 표시하고, 2스테이지 자녀도 보스 종류의 축소 아이콘으로 표시.

- 레이스 화살 장전 속도가 너무 느림
  - 원인: 레이스 전용 화살 쿨다운이 2초 기준으로 유지됨.
  - 조치: 화살 쿨다운을 1초로 낮추고 HUD/무기 설명 문구를 함께 수정.

- 2스테이지 보스 패턴이 부족함
  - 원인: 보스 HP 절반 이하 상태에서 자녀 소환, 추격 에너지볼, 레이저 공격이 없음.
  - 조치: 2스테이지 진입 후 자녀 소환, 플레이어 추격, 에너지볼 발사, 레이저 경고/타격 판정을 추가.

- 스타홀 투어 열기구가 실제로 타지지 않음
  - 원인: 열기구 오브젝트만 경로를 따라 움직이고 플레이어 탑승/동기화 상태가 없음.
  - 조치: 투어 중 플레이어를 열기구 바구니 좌표로 이동시키고, 임시 열기구 마운트를 장착한 뒤 종료 시 이전 탑승 상태로 복구.

- 날개가 등에 붙어 보이지 않고 천사 날개가 단순함
  - 원인: 날개 렌더 함수가 캐릭터 X 좌표를 받지 않아 캐릭터 중심이 아닌 원점 기준으로 그려짐.
  - 조치: 날개 렌더 좌표를 캐릭터 중심에 고정하고, 천사 날개를 큰 깃털 레이어 형태로 재디자인.
